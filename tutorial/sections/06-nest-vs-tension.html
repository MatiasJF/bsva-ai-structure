Nest or tension?

Every substantive task becomes a Nestr node. Which kind depends on one question: are you crossing a role boundary?

For own work
Nest
Your role is accountable. You don't need anyone else's time or authority.
  • Past-tense title ("Blog on BRC-100 published")
  • Acceptance criteria in the description
  • users: [you] required
  • Progress posted as comments
For cross-role asks
Tension
Another role's decision, time, or approval is needed to finish.
  • One of 5 pathways (info / outcome / action / governance)
  • Request body is clear and self-contained
  • Cluster related asks into a single tension
  • Submit in batches (≤ 2/day/target role)

The test — one question

Rule of thumb

Does completing this require anyone else's calendar time or authority? → Tension. Otherwise → Nest.

Three concrete examples

DevRel writes a blog
Nest (yours). + Tension to Tech's sdk-maintainer for the crypto section's fact-check.
Tech refactors a service
Nest (yours). + Tension to Security — hard rule for crypto/consensus paths.
Biz-Dev builds a lead list
Nest (yours). No tensions — public sources, no one else's time needed.
Nestr norm — don't spam tensions

An agent creating 20 tensions per hour is technically correct but practically hostile. Target ≤ 2 submissions per day per target role. Use the nestr-cluster skill to batch related asks into one well-structured tension.
