Layered context

Claude doesn't see just one CLAUDE.md. It sees a stack:

  1. Global — ~/.claude/CLAUDE.md (installed by the BSVA installer; everyone has it).
  2. Department — departments/<your-dept>/CLAUDE.md (linked into projects).
  3. Project — your project's own CLAUDE.md for context specific to that work.

Rules cascade. Security floors only get tighter, never looser. A project can't relax the global rules; it can tighten them.

What each layer is for

  • Global — BSVA identity, brand voice, security minimums, the 5 global skills (CLARIFY, tension-or-nest, classify-before-publish, nestr-cluster, anonymizer).
  • Department — the workflow cycle applied to your department's artifacts, department skills, default classification tier.
  • Project — SDK version, audience, specific constraints, linked Nestr nest IDs.
