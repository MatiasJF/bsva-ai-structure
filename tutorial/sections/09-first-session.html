Your first real session

Walk through the cycle once with a concrete example. Copy the prompts, paste them into Claude, and watch the discipline pay off.

Pick a small real task

For your first session, pick something you'd actually do this week. It should be:

  • Shared (not throwaway scratch) — so the cycle applies.
  • Contained (1–2 hours of work) — so you can close the loop today.
  • Not Restricted — so you can complete it without legal/exec escalation.

Example: writing a short blog post, drafting a lesson outline, refactoring a small internal tool, building a lead list for a specific vertical.

Step-by-step

Step 1 — Start Claude in the project folder

cd ~/projects/my-new-task
claude

Step 2 — Open with a grounding prompt

Tell Claude what you want to do. Don't jump to "write me X" — describe the desire.

I want to [your task, one sentence]. Before we start, walk me through a CLARIFY round — what do you need from me?

Claude invokes the clarify skill. It asks you the universal 5 plus artifact-specific extras. Answer them all in one batched reply.

Step 3 — Let Claude propose the Nestr capture

Good. Now propose the Nestr node — nest or tension? Show me the title, acceptance criteria, and labels before creating.

Claude runs tension-or-nest. Review what it proposes. If it looks right, say "go" — Claude creates the node via the Nestr MCP and returns the ID.

Step 4 — Enter plan-mode

Now enter plan-mode and draft the execution plan. Reference the nest ID.

Or press shift+tab in the prompt. Claude drafts a plan with files, approach, dependencies, estimate, risks.

Step 5 — Push back

Read the plan. Push back on anything. Trim scope. Identify dependencies. Iterate for as many rounds as needed.

Step 6 — Approve explicitly

Approved. Go.

Step 7–8 — Claude scaffolds + executes

Claude copies the right template, inits git, starts working. Post progress comments to the Nestr nest at each milestone.

Step 9 — Report on close

Work is done. Write the REPORT comment on the Nestr nest — outputs, plan deltas, any new tensions spawned, promotable learnings.

Step 10 — Close the loop

You (not Claude) review acceptance criteria. If met, set project.status: Done, completed: true. Congratulations — you just ran a full cycle.

What you should feel afterwards

  • Mild friction in the first 2 steps (CLARIFY feels slow) — that disappears by week two.
  • Surprise that plan-mode caught a scope issue.
  • The Nestr nest has a nicely structured description you can re-use as a template later.
  • You saved a prompt you re-typed. Candidate for a skill — see the retro file in the repo.
If something slipped

If you realize you pasted something you shouldn't have, or the reply included a detail that should have been Confidential, go straight to security/incident-response.md. Blameless reporting is how BSVA stays safe.
