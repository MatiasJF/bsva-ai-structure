What just happened

A quick recap of what the installer did — so you know what changed on your machine and how to verify it.

The installer's job

install.sh (or install.ps1 on Windows) is idempotent — safe to re-run. It:

  1. Created ~/.claude/ if it didn't exist.
  2. Copied the global skills into ~/.claude/skills/.
  3. Installed the base CLAUDE.md at ~/.claude/CLAUDE.md — Claude reads this every session.
  4. Wrote MCP and settings templates you can merge into your config.
  5. Asked which department you're in (recorded at ~/.claude/.bsva-department).
  6. Backed up anything it replaced to ~/.claude/.bsva-backup/<timestamp>/.

Three things to do now

1. Add your Nestr API key to your MCP config

Open ~/.claude/mcp.bsva-template.json. Replace <your-nestr-api-key> with the key from Nestr → Settings → API. Merge the mcpServers block into your real ~/.claude/mcp.json (or into the Desktop app's MCP settings). Restart Claude.

2. Review the settings template

Open ~/.claude/settings.bsva-template.json. Don't blindly overwrite your personal settings.json — copy the permission allow/deny blocks across, and keep your own stricter rules.

3. Verify

Open Claude and ask:

Use the Nestr MCP to tell me who I am and which workspace I am in.

You should see your name, role, and workspace. If you see an auth error, your API key didn't land in the right place.

To re-run the installer

Run ./install.sh again from the repo folder. Add --sync to skip the department prompt. The installer will back up anything it replaces.

Keeping up to date

When the repo changes, pull and sync:

cd ~/bsva-ai-structure
git pull
./install.sh --sync
