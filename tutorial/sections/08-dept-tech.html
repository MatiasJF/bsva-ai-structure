Key skills (installed for Tech)

  • tech:new-service — scaffolds a new backend service with BSVA conventions (.env ignored, design doc, CI baseline).
  • tech:new-rfc — emits BRC-shaped tension parts with BSV Academy cross-references.
  • tech:review-pr — runs the BSVA review checklist against a diff; flags hallucinated symbols and unasked refactors.
  • tech:incident-draft — structures a Confidential postmortem; refuses unredacted logs.
  • tech:research-note — template-driven spike, forces question + decision + method + "did not do".
  • tech:classify-diff — scans a diff for secrets / customer identifiers / production hostnames before commit.

Tech-specific rules

Hard lines
  • Never production signing. Testnet only if explicitly scoped.
  • Never HSM configuration, policies, or credentials in a Claude session.
  • Never raw customer data — redact or use synthetic fixtures.
  • Never add dependencies without verifying against the registry (kills supply-chain drift).
  • Crypto / auth / consensus code → second reviewer from Security required.
