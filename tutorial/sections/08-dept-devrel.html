Key skills (installed for DevRel)

  • verify-sdk-surface — kills hallucinated @bsv/sdk methods. Run on every SDK snippet, no exceptions.
  • sdk-example-scaffold — scaffolds a minimal SDK example. Enforces "one concept per example".
  • draft-dev-support-reply — drafts a support reply + triage decision (answer / repro-needed / bug-to-Tech / known-issue).
  • workshop-lesson-splitter — turns a workshop outline into lesson branches with starter + solution.
  • blog-post-from-transcript — converts a talk transcript into a BSVA-voice blog draft.
  • triage-sdk-issue — classifies a dev-support thread in one shot.

DevRel-specific rules

  • Never publish examples with hardcoded private keys or reused testnet keys.
  • Never write about unreleased SDK features without sign-off.
  • Never paste a developer's real user code — demand a sanitized minimal repro.
  • Always run verify-sdk-surface before any SDK code leaves your machine.

Escalation owners

  • Tooling / structure — DevRel Lead.
  • SDK correctness — sdk-maintainer (Tech).
  • Partner naming — Biz-Dev Lead (before publish).
  • Licensing — Legal Lead (before merge).
