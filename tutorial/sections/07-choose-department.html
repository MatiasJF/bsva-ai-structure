Choose your department

The next section of this tour is department-specific. Pick the one you work in — we'll remember it for later, and you can always come back to this page to switch.

Developer Relations
SDK support, examples, workshops, blog, events

Default tier: Internal → Public on publish.
Technology / Engineering
Services, RFCs, research, code review, incidents

Default tier: Internal. Confidential for security work.
Education / BSV Academy
Lessons, assessments, translations, learner support

Default tier: Internal drafts → Public on publish. Confidential for learner data.
Operations
Biz-dev, marketing, legal, exec

Default tier: Confidential (biz-dev). Restricted (legal + exec).
What if I work across departments?

Pick whichever covers most of your work. Each Claude session is usually scoped to one project, and each project links one department's CLAUDE.md. Switch departments by re-running ./install.sh or editing ~/.claude/.bsva-department.
