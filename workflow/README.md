# Workflow

This folder defines **how BSVA works with Claude**. Not where files live (that's `departments/`), but the **discipline every session follows**.

The short version:

```
DESIRE → CLARIFY → CAPTURE → PLAN-MODE → DISCUSS → APPROVE → SCAFFOLD → EXECUTE → REPORT → CLOSE
```

Every department applies the same cycle to different artifacts. Read the files in order.

---

## When to follow the full cycle

**Mandatory for shared / shipped work.** Anything that:
- Goes external (blog, talk, repo, partner communication).
- Lands in a shared BSVA system (SDK, Academy, Nestr tensions, main branches).
- Takes more than 30 minutes or touches more than 2 files.
- Requires review by another role.

**Optional for personal scratch.** Throwaway prototypes, personal notes, one-shot research — do the cycle if it helps, skip if it's ceremony.

The distinction: **does the output have an audience beyond you?**

## The one rule we want everyone to internalize

**Don't act on step 1 alone.** When a colleague or your own future self asks "why did we do it this way", the answer should live in Nestr — not in your head, not in a chat log, not in your DMs. The cycle exists to make "why" visible and durable.
