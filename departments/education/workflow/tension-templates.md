# Education — tension templates

---

## Tension → DevRel (technical review)

**When:** lesson makes protocol / consensus / SDK claims; assessment item involves SDK code; workshop design.

**Title examples:**
- `Technical review of UTXO lesson before publish`
- `Code-read assessment items for SPV lesson — SDK accuracy check`

**Pathway:** 3 (outcome-request).

---

## Tension → Tech (deeper protocol review)

**When:** spec-level questions; consensus-critical claims; BRC-aligned content.

**Title example:** `Accuracy review: is "the Merkle root uniquely commits to the ordered transaction set" a correct framing for beginners?`

**Pathway:** 1 (info request) or 3.

---

## Tension → native-speaker reviewer (translation)

**When:** every translation before publish.

**Title example:** `ES translation review: UTXO lesson v2.1`

**Pathway:** 3 — reviewer's time is required.

Body includes: source ID + version, target register, termbase changes proposed, uncertain terms flagged.

---

## Tension → Education Lead (publication approval / escalation)

**When:** content involves edge cases (minors, sensitive data, jurisdictional concerns); publish approvals.

**Pathway:** 3 (approval) or governance (if structural).

---

## Tension → Legal (PII / minors / jurisdiction)

**When:** any learner-data analysis; minors involved; accommodation-data handling.

**Pathway:** 3.

Restricted tier by default.

---

## Tension → Marketing

**When:** promoting a new course / lesson / module publicly; collateral for enrolment drives.

Cluster with Marketing's overall campaign planning.

---

## Clustering norms for Education

- **Technical-review asks to DevRel**: cluster weekly — one tension listing all lessons needing review.
- **Native-speaker review for translations**: per translator, per week.
- **Lesson updates for a single module**: one tension to update the module nest with all lesson changes.

---

## Governance tensions

Examples:
- `Propose role: Senior Instructional Designer (Education circle)`
- `Propose policy: all new lessons require fact-check-gate pass before publish`
- `Propose domain: BSVA Academy termbase`

---

## Never via tension

- **Learner identifiers** — never in a tension title or body. Use placeholders.
- **Individual learner performance** — belongs in LMS / Education Lead's confidential circle, not in general tensions.
- **Retroactive "I already translated this, now reviewing"** — if it's done, nest it.
