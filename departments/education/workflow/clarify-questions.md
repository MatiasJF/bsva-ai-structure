# Education — CLARIFY question bank

Universal 5 + Education-specific extras per artifact.

---

## Lesson

- **Audience** — absolute-beginner / beginner-dev / intermediate / enterprise-architect / non-technical?
- **Prerequisites** — what must the learner already know? (Name concepts concretely.)
- **Learning objectives** — 3–5 verbs ("identify, compute, distinguish from account model").
- **Length / format** — single page / multi-section / with diagrams?
- **Surface** — Academy LMS / blog / handout / open web?
- **Assessment strategy** — MCQ / short-answer / hands-on / code-read?
- **Primary sources** — BSV Academy concept IDs, BRC references, spec sections?
- **Reading level target** — explicit grade level / jargon-density?

## Assessment item

- Which lesson(s) does this assess?
- Item types — MCQ / multi-select / short-answer / code-read?
- Bloom level per objective (remember, understand, apply, analyze, evaluate, create)?
- How many items per objective?
- **Distractor strategy** — plausible misconceptions, not nonsense distractors.
- Passing threshold?

## Translation

- Source asset ID + version (translations stale the moment EN changes).
- Target register — formal `usted` / neutral / LatAm vs Iberian / Traditional vs Simplified / Brazilian vs Portugal Portuguese?
- Glossary / termbase in use?
- Native-speaker practitioner reviewer identified?
- Same translator edits + reviews, or separate reviewer required?

## Learner-support reply

- Is this PII-laden? (Redact before proceeding with `anonymizer`.)
- Public channel (forum, LMS comment) or private (email, ticket)?
- What's the learner actually stuck on (diagnosis)?
- Is there a documentation gap this reply exposes?
- Should the reply draft be generic (for others too) or learner-specific?

## Course module

- Audience + prerequisites at module level?
- Terminal objectives — what can a learner do after completing the module?
- Lesson map — list, with each lesson's 1-line objective?
- Assessment strategy at module level (capstone, certification)?
- Estimated total time?

---

## Education-specific universal-5 phrasing

> "Before I start on this lesson, let me ground it.
>
> - **What** — concretely: an Academy lesson on UTXOs, single-page, beginner-dev audience?
> - **Why** — new lesson for a module gap, or refresh of an existing one?
> - **Who for** — beginner-dev (some programming but no Bitcoin background)? Or a non-technical learner?
> - **Success** — lesson published + 3 checks-for-understanding + peer-reviewed + technical-reviewed by DevRel?
> - **Tier** — Internal → Public on publish.
>
> Plus pedagogy: prerequisites, 3–5 learning objectives, reading level target, assessment strategy."

## Skipping CLARIFY (Education edge cases)

- Minor edit to my own published lesson → skip CLARIFY, note the edit in the nest's comments.
- Spell-check or format fix → skip.
- Answering a public forum Q where no PII is involved → abbreviated CLARIFY (just confirm the diagnosis).

Never skip for: new lessons, new assessments, translations, learner-support with PII, course-module design.
