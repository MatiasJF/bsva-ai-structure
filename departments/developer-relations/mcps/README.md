# DevRel department MCPs

MCPs scoped to DevRel work. Most of what DevRel needs is already in the global set (Nestr, simple-mcp, WhatsOnChain, BSV Academy). Add MCPs here when they're DevRel-specific.

## Adding an MCP

See `guides/visuals/04-mcp-lifecycle.md` for the full flow. Summary:

1. Check that no existing MCP covers the use case.
2. Draft a proposal (what / why / scope / credentials / classification).
3. Security + DevRel review.
4. Add config entry here with a clear `_comment` field.
5. Add to `security/approved-external-tools.md` at the appropriate tier.
